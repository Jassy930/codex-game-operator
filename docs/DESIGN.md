# Design

核心循环：点击获得资源 -> 购买升级 -> 提升产出 -> 追逐下一目标。

M0 设计约束：

- 第一次升级成本为 10，确保短时间内出现明确反馈。
- 第八次连续点击触发过载奖励，界面必须显示过载进度和剩余点击数，增强早期节奏。
- 过载奖励必须有可见成长线：星核谐振器提升过载奖励，顶部状态区必须显示当前过载奖励。
- 点击、过载和购买升级后必须显示即时行动结果，说明本次收益或产能变化。
- 目标条按“首次升级 -> 自动产能 -> 累计能量 -> 稳定器”推进，并必须显示进度、剩余量、购买型目标的关键升级能量缺口和完成状态。
- 购买型目标对应的升级卡片必须显示“目标推荐”并高亮，降低目标条和升级区之间的匹配成本。
- 100K 以后必须出现可见的中后段项目目标；下一个未完成项目必须进入主目标条，项目目标要显示进度、剩余量、完成状态和奖励，主目标条也要直接显示当前项目奖励，项目完成提示必须确认已获得的奖励，并对有效点击/自动/总产能或过载奖励产生可测试影响。
- 星图主目标、星图总览和星图项目列表必须显示航段序号，让玩家知道当前项目位于完整星图路线的第几段。
- 点亮星图后的星图项目必须包含谐振校准目标，让中后段内容不只围绕常规产能倍率循环。
- 累计 100K 能量后必须解锁星图航线策略，让玩家能在点击、自动和默认节奏之间主动切换；旧存档默认策略不得改变既有产能。
- 星图航线策略后必须提供策略专精航段：点火航校强化点击路线，巡航航校强化自动路线，均衡校准强化总产能路线。
- 航线策略按钮在解锁后必须显示对应专精航段进度或完成状态，让策略选择和可追目标直接关联。
- 星图总览必须按当前航线策略显示对应专精航段、奖励和进度，让策略选择和中后段目标在同一信息层级可见。
- 星图计划必须提供总览，显示已完成项目数、总项目数、下一段名称、下一段奖励、进度详情、目标分轨、阶段导航、章节目标、章节奖励、奖励进度、奖励目标、里程碑、航线焦点、航线构成、奖励分布、已生效星图加成和最多 3 个未完成项目的名称与奖励预告；目标分轨必须同时显示下一个累计航段和下一个升级航段；阶段导航必须显示首段星图、专精校准、深空基建、远航长尾四个阶段的完成数、当前阶段、本阶段剩余航段数和下一阶段名称；章节目标必须显示四个章节各自下一段未完成目标，章节完成后显示“已完成”；章节奖励必须按四个章节汇总总产能、点击、自动和过载奖励数量；奖励进度必须按总产能、点击、自动和过载显示已生效奖励数/总奖励数；奖励目标必须按总产能、点击、自动和过载显示下一条未完成奖励航段、奖励内容和当前进度；里程碑必须显示当前章节终点和终局航点的航段序号、奖励与进度，全部完成后显示全部章节已完成；航线焦点必须按当前航线策略显示对应专精航段、奖励和进度；航线构成必须显示累计/升级航段数量和四类奖励分布；全部完成时必须显示完成状态。
- 星图总览必须显示行动建议：升级航段复用升级可购买/能量缺口文案，累计航段显示剩余能量，当前自动产能大于 0 时显示预计时间；行动建议只影响展示，不改变存档、航段顺序、奖励数值和升级价格。
- 星图项目列表必须显示每个项目所属章节、章节内位置、推进方式标签、奖励类型标签和状态标签；状态标签必须区分“已完成”“当前航段”和“待推进”，并且只有第一个未完成项目显示为“当前航段”；全部完成时不得显示当前航段。
- 星图项目列表必须提供全部、当前、本章、累计、升级、总产能、点击、自动、过载、未完成和已完成筛选；本章筛选只显示当前航段所属章节，全部完成时为空；累计/升级筛选按项目是否依赖累计能量或升级等级归类；奖励类型筛选只显示对应成长线的航段；全部、本章、累计、升级和奖励类型等内容筛选按钮必须显示已完成/总数，当前、未完成和已完成筛选按钮必须显示直接数量；筛选区必须显示当前视图的航段数、完成数、完成率、剩余航段数、状态构成、奖励构成、推进构成、章节构成、已领取奖励、待领取奖励、下一条可推进航段、后续最多两条未完成航段和筛选终点；已领取奖励只在当前视图同时存在已完成和未完成航段时显示，避免和已完成筛选的奖励构成重复；章节构成按首段星图、专精校准、深空基建、远航长尾顺序统计当前筛选视图中的命中航段；筛选只改变列表展示，不改变存档、项目完成判定、奖励数值或航段顺序。
- 250K 以后星图计划必须继续提供至少一个长尾项目目标，避免首批项目完成后只剩循环累计目标。
