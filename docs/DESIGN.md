# Design

核心循环：点击获得资源 -> 购买升级 -> 提升产出 -> 追逐下一目标。

M0 设计约束：

- 第一次升级成本为 10，确保短时间内出现明确反馈。
- 主操作区必须包含随静态构建发布的本地视觉资产，表现星核和关键升级装置；视觉资产只改变展示，不改变存档、数值、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供即时视觉反馈：普通点击触发短脉冲、粒子层、点击落点闪光和本次 `+X` 收益浮层，接近过载时显示蓄能前兆，过载触发时显示更强冲击波、过载落点闪光和过载收益浮层；落点闪光只从点击事件坐标派生，键盘触发必须回退到按钮中心；反馈只改变展示，不改变存档、点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供可关闭的本地音效反馈：默认开启，普通点火与过载点火使用不同 Web Audio 短音；偏好只能写入独立 localStorage key，不进入游戏进度存档，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供可关闭的移动端触感反馈：默认开启，普通点火使用短震动，过载点火使用更强节奏；偏好只能写入独立 localStorage key，不进入游戏进度存档，触感调用必须通过浏览器 `navigator.vibrate` 能力检测，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供环形蓄能轨：由现有连击进度派生 `--core-charge-angle`，普通连击逐步点亮，过载前兆和过载命中时高亮；蓄能轨只改变展示，不新增存档字段，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供阶段光环：由现有连击进度派生 `--core-stage-angle`，3/8 后预热、6/8 后升温、过载前一击脉动，过载命中时爆发；阶段光环只改变展示，不新增存档字段，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须提供过载倒计时徽标：由现有 `getComboStatus()` 派生剩余过载次数，过载前一击高亮，过载命中时显示爆发态；倒计时徽标只改变展示，不新增存档字段，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 主点火按钮必须显示 8 格连击轨和下一击奖励预告，直接展示当前连击进度、下一次点火收益和临近过载的奖励；预告只从现有点击产能、过载奖励和连击状态推导，不新增存档字段，不改变点击收益、连击窗口、过载奖励、升级价格、星图航段、航线策略、航线指令或反馈入口。
- 第八次连续点击触发过载奖励，界面必须显示过载进度和剩余点击数，增强早期节奏。
- 过载奖励必须有可见成长线：星核谐振器提升过载奖励，顶部状态区必须显示当前过载奖励。
- 点击、过载和购买升级后必须显示即时行动结果，说明本次收益或产能变化。
- 目标条按“首次升级 -> 自动产能 -> 累计能量 -> 稳定器”推进，并必须显示进度、剩余量、购买型目标的关键升级能量缺口和完成状态。
- 购买型目标对应的升级卡片必须显示“目标推荐”并高亮，降低目标条和升级区之间的匹配成本。
- 升级卡片必须显示按升级类型区分的固定尺寸图标，覆盖点击、自动、总产能和过载成长线；图标只改变展示，不改变存档、升级价格、产能公式、星图航段、航线策略、航线指令或反馈入口。
- 100K 以后必须出现可见的中后段项目目标；下一个未完成项目必须进入主目标条，项目目标要显示进度、剩余量、完成状态和奖励，主目标条也要直接显示当前项目奖励，项目完成提示必须确认已获得的奖励，并对有效点击/自动/总产能或过载奖励产生可测试影响。
- 星图主目标、星图总览和星图项目列表必须显示航段序号，让玩家知道当前项目位于完整星图路线的第几段。
- 点亮星图后的星图项目必须包含谐振校准目标，让中后段内容不只围绕常规产能倍率循环。
- 累计 100K 能量后必须解锁星图航线策略，让玩家能在点击、自动和默认节奏之间主动切换；旧存档默认策略不得改变既有产能。
- 航线策略按钮必须提供固定尺寸视觉徽记，覆盖均衡航线、点火优先和巡航优先三种策略；徽记只改变展示，不新增存档字段，不改变航线策略倍率、解锁条件、专精航段、航线指令、星图路线、升级价格或反馈入口。
- 累计 100K 能量后必须出现航线指令，让玩家在点火、等待自动产能和购买升级之外有主动选择；首批指令必须分别对应点击、自动和过载收益，并用独立冷却限制爆发频率。航线指令必须支持 90 秒“航线连携”：轮换不同指令给下一次指令 +12%/+24% 封顶加成，完成 3/3 轮换时额外获得基础指令收益 18% 的轮换目标奖励；如果 3/3 第三步收束到当前航线策略契合指令，还必须额外获得基础指令收益 12% 的策略终结奖励；重复同类或超时重置；旧存档必须自动补齐指令冷却和连携字段；指令不得改变升级价格、星图航段顺序、项目完成判定、项目奖励或筛选规则。
- 航线指令按钮必须提供固定尺寸视觉徽记，覆盖点火齐射、巡航回收和谐振脉冲三种主动指令；徽记只改变展示，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度、星图路线、升级价格或反馈入口。
- 航线指令区必须包含项目内本地视觉插画，表现点火齐射、巡航回收、谐振脉冲和 3 步轮换轨；插画只改变展示，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度、星图路线、升级价格或反馈入口。
- 航线指令徽记必须提供状态灯：锁定态显示默认弱状态，可执行态显示可用状态，冷却中显示冷却状态，推荐态可加强状态边框；状态灯只从 `ready`、`cooling`、`recommended` 派生，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度、星图路线、升级价格或反馈入口。
- 航线指令按钮必须压缩徽标和收益明细的默认可见文本：运行期每个按钮最多直接显示 3 个文字徽标，其余用 `+N` 提示；预计收益行显示“预计 +X 能量 · N 项明细”，完整 `previewText` 必须保留到 `title` / `aria-label`。该压缩只改变展示层，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度、星图路线、升级价格或反馈入口。
- 航线策略必须和航线指令形成主动选择关系：均衡航线契合谐振脉冲，点火优先契合点火齐射，巡航优先契合巡航回收；匹配当前策略的指令获得 10% 策略契合加成，并在预计收益、执行反馈、按钮徽标和本地 `directive` 事件中可见。策略契合不得新增存档字段，不得改变指令冷却、连携窗口、轮换目标奖励、星图航段或升级价格。
- 航线指令必须显示可执行的轮换目标：锁定时提示 100K 解锁；解锁后显示 0/3 起步目标，并在 0/3 时把非契合指令标为“收束起手”、把当前策略契合指令留给第三步策略终结，推荐起手仍在冷却时显示“等待起手”；1/3 且上一指令不是当前策略契合指令时，必须继续推荐另一条非契合指令并显示“收束续航/等待续航”，避免第二步提前消耗第三步策略终结指令；连携窗口内显示当前指令、轮换进度、剩余窗口和下一步建议；接近 3/3 时必须提示轮换目标奖励，且当下一步可收束到当前策略契合指令时提示策略终结奖励；推荐的下一步指令必须在按钮层显示“轮换推荐”，推荐但仍冷却的指令显示“等待轮换”，可触发策略终结的推荐指令必须额外显示“策略终结”徽标；该提示只从现有冷却与连携字段推导，不新增存档字段。
- 航线指令轮换目标必须提供 3 格视觉轨：锁定态显示 100K 解锁；解锁后按 `getDirectivePlan.progress` / `target` / `nextDirectiveIds` 显示 1/3 到 3/3 的已完成、下一步和待推进状态，下一步格必须显示当前推荐指令名；完成 3/3 后显示轮换完成。视觉轨只改变展示，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度或星图项目。
- 航线指令支持说明必须默认摘要化：`directivePlan` 显示 `summaryText`，航线委托显示进度/完成短句，远航调度显示当前航段和目标指令短句；完整说明必须保留到 `title` / `aria-label`，并用单行省略避免小屏文字墙。摘要化只改变展示，不新增存档字段，不改变指令收益、冷却、连携窗口、远航调度或星图项目。
- 航线指令推荐预案必须有即时收益反馈：执行当前 `getDirectivePlan.nextDirectiveIds` 推荐的指令时，额外获得有效基础指令收益 6% 的“预案执行”奖励；预计收益、执行反馈和本地 `directive` 事件必须记录 `planReward` / `planBonusRate`；按钮徽标区必须直接显示 `planRewardText`，让推荐指令的额外收益可扫视。预案执行不得新增存档字段，不得改变升级价格、星图航段顺序、项目完成判定、项目奖励、指令基础收益、冷却、连携窗口、轮换目标奖励、策略契合、策略终结、指令熟练、满层回响或筛选规则。
- 航线指令必须提供独立短期任务“航线委托”：累计 100K 能量后显示 0/3 到 3/3 推荐轮换委托，并用固定进度条和 `role="meter"` 暴露当前进度；当玩家按推荐预案把轮换从 2/3 完成到 3/3 时，额外获得有效基础指令收益 8% 的“委托完成”奖励；按钮徽标、预计收益、执行反馈和本地 `directive` 事件必须记录 `taskReward` / `taskRewardRate`；3/3 后必须显示委托已完成，并提示重置或超时后开启下一轮委托。航线委托不得新增存档字段，不得改变升级价格、星图航段顺序、项目完成判定、项目奖励、指令基础收益、冷却、连携窗口、轮换目标奖励、预案执行、策略契合、策略终结、指令熟练、满层回响或筛选规则。
- 20M 后必须提供“远航调度”，让当前未完成星图航段改变目标指令：点击奖励航段对应点火齐射，自动奖励航段对应巡航回收，过载奖励航段对应谐振脉冲，总产能奖励航段对应当前航线策略契合指令；目标指令必须接管指令轮换的下一步推荐并显示“调度目标/等待调度”，当调度目标与常规收束起手冲突时优先服从当前航段调度；执行目标指令必须获得有效基础指令收益 14% 的“调度校准”奖励，目标指令冷却必须按 0.7 倍生效并显示“调度冷却 -30%”，且本轮指令连携窗口必须从 90 秒延长到 120 秒并显示“调度接力 +30 秒”；目标指令后的 1/3 阶段必须把可用非目标指令标为“远航续航/等待续航”，允许任意非目标指令触发有效基础指令收益 8% 的远航续航奖励，同时按目标指令指定一个协同续航指令，该按钮必须单独显示“远航协同/等待协同”，执行该指令额外获得有效基础指令收益 5% 的“远航协同”奖励，并额外获得有效基础指令收益 3% 的“协同补给”当前能量补给；协同补给必须只增加当前能量，不增加累计能量；执行指定协同续航后回到当前航段目标指令时，必须显示“协同回航/等待协同”，并提示触发远航闭环与远航突破；目标后的另一个非目标指令必须显示“远航绕行/等待绕行”，执行时额外获得有效基础指令收益 4% 的“远航绕行”奖励，并触发“绕行投送”：消耗少量当前能量，按 150% 投送量推进累计能量；绕行后下一步回到当前航段目标指令时显示“绕行回航/等待绕行”、额外按当前能量航段剩余量 0.03% 注入“绕行突破”；当 3/3 指令轮换回到当前航段目标指令时，必须额外获得有效基础指令收益 16% 的“远航闭环”奖励，并按当前能量航段剩余量的 0.05% 获得“远航突破”能量注入，同时触发“远航整备”刷新协同续航指令冷却，完成态下一步必须优先推荐“整备续航/等待整备”；执行该整备推荐的协同续航指令必须额外获得有效基础指令收益 7% 的“整备续航”奖励；执行整备续航后回到当前航段目标指令时，必须显示“整备回航/等待回航”，并额外获得有效基础指令收益 6% 的“整备回航”奖励；按钮徽标、预计收益、执行反馈和本地 `directive` 事件必须记录远航协同补给、远航绕行、绕行投送消耗、绕行投送累计推进、绕行突破、远航突破、远航整备、整备续航和整备回航字段。远航调度条必须同时显示当前航段进度和 0/3 到 3/3 的闭环进度，闭环进度必须使用 `role="meter"` 暴露，并在 1/3 时提示优先协同续航指令触发远航协同与协同补给，或切另一非目标指令触发远航绕行并进行绕行投送，在 2/3 时提示协同回航到目标触发远航闭环与远航突破，或在绕行路线回到目标追加绕行突破，在 3/3 时提示远航整备优先触发整备续航，整备续航后提示回到目标触发整备回航；按钮徽标、预计收益、执行反馈、本地 `directive` 事件和反馈快照中必须可见相关状态。除记录最近远航分支的 `farRouteLastBranchDirectiveId` 外，远航调度不得新增其他存档字段，不得改变升级价格、星图航段顺序、项目完成判定、项目奖励、航线策略、指令基础收益、基础连携倍率、轮换目标奖励、预案执行、航线委托、策略契合、策略终结、指令熟练或满层回响。
- 远航绕行路线必须提供“绕行整备”：选择绕行并回到目标完成闭环时，必须刷新刚才的绕行指令冷却；完成态下一步必须优先推荐“绕行整备/等待绕行整备”，执行该绕行指令必须额外获得有效基础指令收益 5%，并在按钮徽标、预计收益、执行反馈、本地 `directive` 事件和反馈快照中记录 `dispatchDetourPrepReward` / `dispatchDetourPrepRewardRate` / `dispatchDetourPrepRewardText`。
- 远航调度必须显示当前分支态势：active 状态下由现有 `directiveChain`、目标指令、协同指令和 `farRouteLastBranchDirectiveId` 派生 `branchKind` / `branchText`，覆盖待选择、协同、绕行、协同整备和绕行整备；主操作区必须渲染 `far-dispatch-branch`，星图总览和反馈快照必须包含分支态势。该展示不新增存档字段，不改变收益、冷却、连携窗口、星图航段、项目奖励、反馈入口或部署链路。
- 远航调度必须提供“分支改道”：目标指令后的分支选择阶段，如果当前选择的协同/绕行分支不同于上一轮远航分支，必须额外获得有效基础指令收益 6%；上一轮协同后改走绕行、上一轮绕行后改走协同触发，重复同一分支不触发。该机制复用 `farRouteLastBranchDirectiveId`，不得新增存档字段；按钮徽标、预计收益、执行反馈、本地 `directive` 事件和反馈快照必须记录 `dispatchBranchShiftReward` / `dispatchBranchShiftRewardRate` / `dispatchBranchShiftRewardText`。
